Operating Systems Technology

Term Definition
Operating System Software packages that oversee and coordinate a computer's internal and external activities and communication
Job Queue A "job" refers to a program execution, therefore a "job queue" refers to the sequence of stored jobs
FIFO "First-In, First-Out" - the standard fashion in which jobs are ordered, that is, the order in which they arrived
Interactive Processing Allows a program to use remote terminals to carry on dialogue with a user
Time-sharing Operating systems that can service multiple users at once
Multiprogramming A time-sharing technique in which time is broken into intervals, with job execution being restricted to a single interval at a time
Application Software Programs meant to perform tasks specific to hardware's utilization
System Software Programs meant to perform geneal tasks common across most, if not all, hardware
Utility Software Programs not included in the operating system, but are necessary for computer installation
Graphicaul User Interface (GUI) A method of user communication in which an operating system uses icons to represent files, programs, and more which the user can then interact with
Shell Older iterations of user interfaces which uses text-based messages to communicate with users
Kernel An internal segment of an operating system that contains software components needed for basic installation functions
Device Driver A software device uniquely designed for the specific peripheral device it is supposed to communicate with
File Manager A piece of software designed to record and manage a device's mass storage facilities
Memory Manager Like the file manager, it is designed to record and manage a computer's main memory
Virtual Memory A large memory space created by paging, which refers to when the memory manager divides data into "pages" that are then shuffles between main memory and mass storage
Boot Strapping A procedure that transfers the main operating system from mass storage to main memory
ROM Read-Only Memory, where the computer collects fundamental software routines for important input and output activities
Deadlock A situation where a set of processes are blocked due to each waiting for a resource dsignated for another
Spooling The technique of holding data for output at a later convenient time
Super User (or Administrator) A title that gives a user special privileges to alter settings of an operating system, modify critical software, adjust privileges, and perform maintenance activities denied to normal users.
Multi-Factor Authentication Authentication procedures that require more than one piece of evidence (i.e. a password) in order to prove that you are who you claim to be
