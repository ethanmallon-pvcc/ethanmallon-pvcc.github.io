All About GitHub

  • Q: What is a version control system?
      • A: A type of database that saves a "snapshot" of a current project then records and displays how it changes.
  • Q: What is Git?
      • A: A distributed version control system typically used by collaborating programmers.
  • Q: What is Github?
      • A: A company that offers a cloud-based Git repository hosting service.
  • Q: What is the difference between Git and GitHub?
      • A: Git is a version control system distributed and hosted by GitHub, hence the name "Git"-"Hub".
  • Q: Who started Github and how was it started?
      • A: The site was launched by Tom Preston-Werner, Chris Wanstrath, P. J. Hyett and Scott Chaco. It was made with the purpose of providing a platform for programmers to host and manage their software projects using Git
  • Q: What company owns it now?
      • A: GitHub is now a subsidiary of Microsoft as of 2018
  • Q: What is the Octocat?
      • A: Octocat is the name of GitHub's mascot.
  • Git/Github Terms
    • Repository - A location or receptacle where data is stored
    • Commit - To carry out or transfer something
    • Fork - To take/copy preexisting source code and use it to create new and distinct code
    • Push - Internet communication where data transfer is initiated buy the publisher or central server
    • Pull Requests - A method of submitted changes to an open developmental project through a distributed VCS
    • Workflows - A program designed to set up, design, and monitor a specific set of tasks and processes with the purposes of increasing overall efficiency and productivity
    • Issues - Anything that is unexpected or prevents things from functioning properly or at all
    • Raw Button - A visual control element which allows the user to initiate an "event"
    • Blame Button - A GitHub feature that allows you to view the commit history of a file
