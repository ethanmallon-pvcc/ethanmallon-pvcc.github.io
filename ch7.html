Chapter 7 Terms

Term Definition
Software Engineering The development of large and complex software systems
Integrated Development Environment (IDE) Systems that combine tools for developing software into a single package.
Association for Computing Machinery
1. Purpose To advance art, science, engineering, and the application of computing by fostering the exchange of information
2. Benefits Networking opportunities, access to a wide array of information, discounts, subscriptions,
3. Student Membership Yes, limited access to benefits
Institute of Electrical and Electronics Engineers (IEEE)
1. Purpose To advance technology for humanity through the inspiration of a global community via a myriad of publications, standards, and activities
2. Benefits Networking, establishment of a professional profile, ability to participate in technical discussions, creation of collaborative and shared project groups, learn about IEEE activities
3. Student Membership Yes, available for a significant discount
Software Life Cycle: Development Phase
1. Requirements Analysis Goal is to specify what services the system will provide, identify constraints on those services, and define outside interaction with the system
2. Design Involves planning system construction, centered around solution development
3. Implementation The actual writing of programs, creation of data files, ad database development
4. Testing The process of debugging and ensuring software compatability. Whether or not this is considered it's own unique step is up for debate as testing constantly occurs during software development
Modular Implementation The development of software through the use of modules, which subdivide and deal with only one part of the softwares responsability
Unified Modeling Language (UML) A modern collection of tools developed with the object-oriented paradigm in mind
Use Case The term used to referred to expected interactions between users and system
Use Case Diagram A UML tool meants to capture the image of a proposed system from user POVs
Data Relationships
1. one-to-one A relationship where one piece of data is tied to only one other peice of data
2. one-to-many A relationship where one piece of data ties to multiple other pieces of data, with those tying back to the first piece exclusively
3. many-to-many A relationship where pieces of data can be associated with multiple other pieces of data without exclusion
Design Pattern a predeveloped model for solving a recurring problem in software design
Software Quality Assurance (SQA) Groups charged with overseeing and enforcing the quality control systems of an organization
Software Testing
1. Glass Box Tester is aware of the interior structure and uses this when designing the test
2. Black Box Tests do not rely on knowledge of interior composition
3. Beta A form of black boz testing done prior to the final version of the product being solidified and released with a segment of the software being available to the audience with the goal of learning how it performs
4. Alpha Like Beta Testing, except performed at the developers site, not by the audience
Documentation
1. User Documentation Explains software features and describes how to use them
2. System Documentation Describes the software's internal composition for later maintenance
3. Technical Documentation Describes how a software system should be installed and serviced
Intellectual Property law
1. Software License A document that expresses the rights of the developer in legal terms
2. Software Disclaimer Used to state liability limitations in licenses
3. Defenses Against Software Liability Claims To apply sound software engineering principles during development, to use a compatible level of care during application, and to produce and maintain validating records.
