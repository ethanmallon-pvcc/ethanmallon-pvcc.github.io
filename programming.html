Chapter 5: Programming Algorithms & Software Testing

  • Phases of computer program development:
    • Phase 1: Understand the problem
    • Phase 2: Design an algorithm to solve the problem
    • Phase 3: Write the program code in appropriate language based on the algorithm
    • Phase 4: Enter the program and save
    • Phase 5: Execute the program
    • Phase 6: Test the program, find errors and debug, then retest
    • Phase 7: Document and evaluate the program
  • Programming structures:
    • Sequential structure: Top-down execution
    • Decision structures: if/then/else
    • Iterative (looping):
      • while loop: continue looping until something is true
      • repeat/until loop: continue looping as long as something is true
      • for loop: loop a given number of times
    • Recursive structures:
      • Repeat a set of instructions as a subtask of itself
      • Repitition continues until:
        • degenerative case, which is a termination condition is met (degenerative case is also called a "base case")
  • Two classic programming/database problems
    • Sorting: Descending order/Ascending order
      • Sorting numbers
      • Sorting strings (in accordance with ASCII values)
    • Searching: find specific data that meet a certain criteria
      • in database:
        • Filters: temporary search
        • Queries: search criteria are saved so the don't have to be recreated
    • Two popular search techniques:
      • Sequential search: search from top to bottom or bottom to top, one item at a time
      • Binary search:
        • only works if data is SORTED first
        • start in the middle, then go to half of that, etc.
  • Two classic program goals:
    • Efficiency: Program takes minimal time and/or resources
    • Correctness: Program runs in expected ways and works for all expected conditions
  • Software testing and verification
    • Kinds of tests:
      • Acceptance testing: system verification to ensure the system works as intended
      • Integration testing: tests to ensure sofware components and/or functions work together
      • Unit testing: tests each software units performance to ensure it works as expected
      • Functional testing: testing via emulating the scenarios the program was designed for
      • Performance testing: testing by giving the software different workloads
      • Regression testing: checking whether the addition of new features break or degrade functionality
      • Stress testing: finding how much strain a system can withstand before failure
      • Usability testing: validation of how well a customer can use the program for the relevant task
    • Software testing is done to detect:
      • Architectural flaws
      • Poor design decisions
      • Invalid or incorrect functionality
      • Security vulnerabilities
      • Scalability issues
    • Testing best practices:
      • Continuous testing: teams test each build as it becomes available, allowing for earlier testing and thus more time to fix issues
      • Configuration management: organizations centrally maintain assets and track software builds. Temas are given access to these assets as necessary
      • Service virtualization: simualtes the services and systmes not yet completed, allowing for earlier testing
      • Defect or "bug tracking": Automated tools allow teams to track defects, understand their scope and impact, and find related problems
      • Metrics and reporting: analytics allowing team members to share status, goals, and results, allowing for the monitor of project health and element relationships
